How to Check Tags

A practical guide for buyers and planners in digital advertising. By the end of this lesson, you will be able to tell whether a campaign tag is working — before the campaign goes live.

Why This Matters to You

As a buyer or planner, you rely on tag data for:

  • Conversion reporting — did the campaign drive real business results?
  • Audience building — remarketing lists only populate if tags fire.
  • Optimisation signals — algorithms need clean conversion data to bid effectively.
  • Invoicing confidence — you need to be certain tracked events reflect real actions.
⚠

A campaign running without a validated tag is effectively blind. You are spending budget with no measurement. Always validate before launch.

Pre-Requisites

You do not need to write code. You need:

  • Google Chrome browser
  • Access to the campaign platform (Google Ads / Meta Ads Manager / TikTok Ads / LINE Ads)
  • The URL of the client's website or landing page

What Is a Tag?

A tag is a small piece of JavaScript code — also called a pixel, snippet, or script — that a platform places on a website to collect data. The tag sends information back to the ad platform every time a visitor does something.

The tag fires silently in the background. The visitor never sees it. The platform receives the event data within milliseconds.

What a Tag Looks Like

Here is what a Meta Pixel base tag looks like in raw HTML. The client's developer (or GTM) places this in the <head> of each page.

<!-- Meta Pixel Code --> <script> !function(f,b,e,v,n,t,s) { if(f.fbq)return; n=f.fbq=function(){...}; ... fbq('init', '1234567890'); // ← your Pixel ID fbq('track', 'PageView'); }(window, ...); </script> <noscript><img height="1" width="1" src="https://www.facebook.com/tr?id=1234567890&ev=PageView"/></noscript>
ℹ

Every tag has a unique ID — the Pixel ID, Conversion ID, or Property ID. This is what links the tag back to your account. Always verify the ID matches your platform account.

Tag vs. Event

People often confuse these two terms.

This is the main tag loaded once across the whole site. It identifies the user and initialises the platform's tracking library. Without this, nothing else works. A GA4 base tag, Meta Pixel base code, TikTok base code, and LINE Tag are all examples.

This fires on top of the base tag for a specific user action — a purchase, a form submission, a button click. For example: fbq('track', 'Purchase') or gtag('event', 'purchase'). These are what feed your conversion reports and bidding signals.

The platform will see page views and can build remarketing audiences. But it will NOT record conversions. Your campaign will report zero conversions even if sales are happening. This is a very common mistake when a client installs the base code but forgets the conversion event.

Tag Generation

Each platform has its own interface for generating a tag. The output is always the same: a snippet of code and an ID. Here is how to generate tags for the four major platforms.

GoogleGoogle Ads Conversion Tag

Google Ads conversion tracking lets you measure specific actions on the website that result from ad clicks.

  1. Go to Goals In Google Ads, click the wrench icon (Tools) → Measurement → Conversions.
  2. Create conversion action Click "+ New conversion action" → select "Website" → enter the website URL → click "Scan".
  3. Set up conversion Name the conversion (e.g., "Purchase"), set value, count (one/every), and click-through window.
  4. Get the tag Select "Use Google Tag Manager" (recommended) or "Install the tag yourself". You will see a Conversion ID (e.g., AW-123456789) and a Conversion Label.
  5. Send to developer or implement via GTM Copy the Conversion ID and Label. Enter them into GTM as a Google Ads Conversion Tracking tag type.
✓

For GA4, the tag is generated in Google Analytics → Admin → Data Streams. Your Measurement ID looks like G-XXXXXXXXXX. Install it via GTM using the "Google Analytics: GA4 Configuration" tag type.

MetaMeta Pixel

The Meta Pixel is a base code that fires on every page. Conversion events are added on top of it.

  1. Open Events Manager Go to Meta Business Suite → Events Manager (or business.facebook.com → Data Sources).
  2. Create or select your Pixel Click "+ Connect data" → Web → Facebook Pixel → "Create Pixel". Give it a name (usually the client name).
  3. Copy the Pixel ID The Pixel ID is a 15–16 digit number shown at the top of the Pixel details page. Keep this safe — this is how all events link back to your ad account.
  4. Get the base code Click "Set up" → "Install code manually". Copy the full base code. It contains your Pixel ID embedded.
  5. Add conversion events After the base code, add standard events like fbq('track', 'Purchase') or custom events on specific pages/buttons. These can also be configured through GTM or the Conversions API.
ℹ

Meta offers the Conversions API (CAPI) as a server-side alternative or supplement to the pixel. This bypasses browser blocking and is increasingly important. Ask your client if they have CAPI set up — if not, tag data may be underreported.

TikTokTikTok Pixel

TikTok's pixel works similarly to Meta. You install a base pixel site-wide, then fire standard events for conversions.

  1. Open TikTok Ads Manager Go to ads.tiktok.com → Assets → Events → Website Pixel.
  2. Create a pixel Click "Create Pixel" → choose "TikTok Pixel" → name it → select your connection method (Pixel code or Partner platform).
  3. Choose installation method Select "Manually install pixel code" or "Use a tag manager". For GTM, TikTok provides direct integration.
  4. Copy the Pixel ID Your Pixel ID appears in the setup page and in Assets → Events. It looks like CXXXXXXXXXXXXXXXXXX.
  5. Set up events Add standard events using the TikTok Event Builder or manually with ttq.track('CompletePayment') for purchases.
⚠

TikTok also offers Events API for server-side tracking. Browser-side pixel data can be limited by iOS privacy changes — check if your client has Events API configured for accurate purchase reporting.

LINELINE Tag

LINE Tag is LINE Ads Platform's tracking pixel. It is widely used in Thailand for LINE advertising campaigns.

  1. Open LINE Ads Platform Go to ads.line.me → select your Ad Account → Conversions → LINE Tag.
  2. Get the Base Code Click "LINE Tag" to see your base code. The Base Code includes your unique Tag ID. Copy it. This goes in the <head> of every page.
  3. Set up Conversion Code Go to "Conversions" → "Create a conversion". Choose the conversion type (Page View, Purchase, etc.). Copy the Conversion Code snippet.
  4. Install via GTM (recommended) In GTM, create a Custom HTML tag. Paste the LINE base code in a tag firing on All Pages. Create a second Custom HTML tag with the conversion code firing only on the confirmation/thank-you page.
  5. Note your Tag ID The Tag ID appears in the base code and looks like a long alphanumeric string. You will need it for validation.
ℹ

LINE Tag does not have a dedicated browser extension. Validation is done via Chrome DevTools (Network tab) or the LINE Ads Platform test conversion tool under Conversions → Tag Setup → Test.

How Tags Fire

Before you can validate a tag, you need to understand what happens technically when a tag fires. This section explains what you are looking for in the browser.

The Three-Step Pattern

Every tag follows the same three-step pattern, regardless of platform:

  1. Trigger condition is met The visitor lands on a page, or clicks a button, or submits a form — whatever was set as the trigger.
  2. JavaScript executes The tag code runs in the browser. It collects data (page URL, event type, order value, etc.) and prepares an HTTP request.
  3. Network request is sent The browser sends a small request to the platform's server — this is the "ping" you can observe in the Network tab of Chrome DevTools.

What You Can See in Chrome DevTools

Open the Network tab in Chrome DevTools (F12 → Network). Each row represents one network request. When a tag fires, you will see a request to that platform's tracking server.

Common Reasons a Tag Does Not Fire

The most common reason. The developer installed the tag on the homepage but not the thank-you or confirmation page where the conversion fires. Check by viewing the page source and searching for the tag ID or domain.

Changes made in GTM are only live after you click Submit and then Publish. If someone updated a tag in GTM but forgot to publish, the live website still runs the old version.

The trigger URL path does not match the actual confirmation page URL. For example, the trigger was set to fire on URLs containing "/thank-you" but the real confirmation page is "/order-complete".

When testing, disable ad blockers and browser extensions like uBlock Origin. These block tracking requests and will make a working tag appear broken. Test in a clean Chrome profile or incognito with extensions disabled.

The tag fires but goes to the wrong account. The Network request will show a 200 (success), but data never arrives in your platform because it was sent to a different account's pixel. Always verify the ID in the network request payload matches your account.

Validate by Platform

Step-by-step validation for each platform. Use both the browser extension method (faster, more visual) and the Network tab method (always works, no install required).

✓

Always test on the actual website, not in a staging environment. Tags configured via GTM may behave differently in staging if GTM has separate environments or if certain URL conditions are set. Test on the production URL before sign-off.

GoogleGA4 + Google Ads Tags

Method A: Google Tag Assistant (Recommended)

  1. Install the extension Search "Google Tag Assistant Legacy" or "Tag Assistant Companion" in the Chrome Web Store. Install it.
  2. Enable recording Click the Tag Assistant icon → click "Enable". Reload the page you want to test.
  3. Read the results Each tag found on the page will show as green (good), yellow (warnings), or red (errors). Click on any tag to see its Measurement ID or Conversion ID.
  4. Check the ID matches your account In Google Ads → Tools → Conversions, or in GA4 Admin → Data Streams, confirm the Measurement ID matches what Tag Assistant found.

Method B: GA4 DebugView (for events)

  1. Install GA4 Debugger extension Search "Google Analytics Debugger" in Chrome Web Store. Enable it.
  2. Open GA4 DebugView In GA4: Admin → DebugView. Keep this tab open while you test.
  3. Trigger the conversion on the website On the client website, complete the action you want to track (e.g., fill out and submit a form). The event should appear in DebugView within 1–2 seconds.
  4. Verify event name and parameters Check the event name (e.g., "purchase") and its parameters (value, currency, transaction_id) appear correctly.

Method C: Chrome DevTools Network Tab

  1. Open DevToolsPress F12 (or right-click → Inspect) → Network tab. Check "Preserve log".
  2. Filter requestsType collect in the filter box at the top of the Network panel.
  3. Reload the pageA request to google-analytics.com/g/collect or analytics.google.com/g/collect should appear.
  4. Click the requestOpen the Payload tab. Confirm tid=G-XXXXXXXXXX matches your GA4 Measurement ID, and en=page_view (or the event name) is present.

MetaMeta Pixel

Method A: Meta Pixel Helper Extension

  1. Install Meta Pixel Helper Search "Meta Pixel Helper" in the Chrome Web Store (published by Meta/Facebook). Install it.
  2. Visit the client website Click the extension icon (the small pixel icon in the toolbar) after the page loads.
  3. Check for a green indicator A green checkmark means the Pixel fired successfully on the page. It will show the Pixel ID and the events that fired (e.g., PageView).
  4. Navigate to the conversion page Complete the conversion flow (e.g., add to cart → checkout → thank you). On the confirmation page, open the extension again and verify the conversion event (e.g., "Purchase") appears.

Method B: Meta Events Manager Test Events

  1. Open Events Manager Go to business.facebook.com → Events Manager → select your Pixel.
  2. Go to Test Events Click "Test Events" tab. Enter the URL of the page you want to test. Click "Open Website".
  3. Trigger the conversion With the test window open, complete the action on the website. Events will appear in real time in Events Manager.
  4. Verify event details Confirm the event name, parameters (value, currency, content_ids), and that no duplicate events are showing.

Method C: Network Tab

  1. Open DevToolsF12 → Network tab. Filter by: facebook.net
  2. Load the pageYou should see a request to connect.facebook.net/en_US/fbevents.js — this means the Pixel code is loading.
  3. Filter by: facebook.com/trThis is the pixel hit endpoint. Each event fires a request here.
  4. Click a request and open PayloadVerify id=YOUR_PIXEL_ID and ev=PageView (or your event name) in the query string.
⚠

If you see "Pixel activated multiple times" in Meta Pixel Helper, the base code is installed twice on the page. This causes duplicate event counts and inflated conversion numbers. Flag this to the developer immediately.

TikTokTikTok Pixel

Method A: TikTok Pixel Helper Extension

  1. Install TikTok Pixel Helper Search "TikTok Pixel Helper" in the Chrome Web Store. Install it.
  2. Visit the client website The extension icon shows a number badge indicating how many TikTok Pixel calls fired on the page.
  3. Click the icon See the Pixel ID, the events fired, and whether any errors exist. Green = success. Red = error.
  4. Test conversion flow Navigate through the full checkout/conversion flow. On the confirmation page, verify the purchase or conversion event fires (e.g., CompletePayment).

Method B: TikTok Ads Manager Test Events

  1. Go to Events in TikTok Ads Manager Assets → Events → select your Pixel → "Test Events" tab.
  2. Enter the test URL Type the URL you want to test. Click "Start Testing". A new tab opens with a tracking cookie active.
  3. Trigger the event In the opened tab, go through the conversion flow. Return to TikTok Ads Manager to see events appear in real time.

Method C: Network Tab

  1. Open DevToolsF12 → Network tab. Filter by: analytics.tiktok
  2. Trigger the page/eventLook for a POST request to analytics.tiktok.com/api/v1/event or a hit to analytics.tiktok.com.
  3. Click the requestOpen Payload. Verify the Pixel ID and event name (e.g., "event":"CompletePayment") in the JSON body.
ℹ

TikTok standard event names to know: ViewContent, AddToCart, InitiateCheckout, CompletePayment, SubmitForm, Contact. Use these exact names — custom event names may not optimise correctly.

GTM Walkthrough

Google Tag Manager (GTM) is the most common way tags are deployed on a website. Instead of editing the website's HTML directly, all tags are managed centrally in GTM and pushed live without a developer code release.

GTM's Three Core Concepts

A Tag only fires when its assigned Trigger condition is met. It reads data from Variables to send accurate information to the platform.

How GTM Gets onto a Website

  1. Developer installs the GTM snippet once Two snippets go into every page of the website: one in the <head> and one immediately after the <body> tag. This is a one-time task for the developer.
  2. All future tags go through GTM The developer never touches the site for tag changes again. You (or the tracking team) manage everything from the GTM interface.
  3. Preview before publishing Use GTM's Preview mode to test tags without affecting the live site. Only after verifying everything works do you publish.

Adding a Tag in GTM — Step by Step

Example: Adding a Meta Pixel base tag.

  1. Go to tagmanager.google.com Open your workspace for the client's container.
  2. Tags → New Click the "New" button in the Tags section of the left sidebar.
  3. Name the tag Use a clear naming convention: [Platform] - [Tag Type] - [Description]. Example: Meta - Pixel - Base Code
  4. Click "Tag Configuration" Choose a tag type. For Meta, there is no built-in type — select "Custom HTML". Paste the Meta Pixel base code into the HTML box.
  5. Click "Triggering" Select "All Pages" — the base pixel should fire on every page. For a conversion tag, create a new trigger: Page URL → contains → /thank-you.
  6. Save the tag Click Save. The tag is saved but NOT yet live.
  7. Preview and test Click Preview (top right). Enter the website URL. GTM Preview mode opens. Navigate the site — you will see which tags fire on each page interaction.
  8. Submit and Publish Once satisfied with Preview testing, click Submit → add a version name (e.g., "Add Meta Pixel") → Publish. The tag is now live.

Using GTM Preview Mode

GTM Preview is your primary testing tool. Here is what to look for:

Status in PreviewMeaningAction
Tags FiredTag fired on this page/eventGood — verify it fired at the right moment
Tags Not FiredTrigger condition was not metCheck the trigger URL/condition matches reality
Tag PausedTag exists but was manually pausedUn-pause if it should be active
⚠

GTM Preview vs. Live. Preview mode uses a special cookie in your browser. Other users visiting the site will NOT see preview traffic. Always test in preview BEFORE publishing to avoid shipping a broken tag to the live environment.

GTM Tag Naming Convention

Consistent naming saves time for everyone on the team. Use this format:

[Platform] - [Tag Type] - [Scope/Event] Examples: GA4 - Config - Base Tag GA4 - Event - Purchase Google Ads - Conversion - Form Submit Meta - Pixel - Base Code Meta - Event - Purchase (Thank You Page) TikTok - Pixel - Base Code TikTok - Event - CompletePayment LINE - Tag - Base Code LINE - Conversion - Purchase

Pre-Launch Validation Checklist

Use this checklist before any campaign goes live. Click each item to mark it as complete. Track your readiness as you go through the checklist.

Before You Start

  • Have the live campaign landing page URL open in Chrome.
  • Disable any ad blockers or open a clean Chrome profile.
  • Have the campaign platform open (Google Ads / Meta / TikTok / LINE).
  • Know the Pixel/Tag/Conversion IDs you expect to see.

Tag Installation

0 of 5 complete

  • Base tag / pixel code is present on the homepage (check Network tab or Tag Assistant)
  • Base tag is present on the campaign landing page (not just homepage)
  • Conversion tag is present on the thank-you / confirmation page
  • Tag IDs in the code match the platform account IDs (no copy/paste errors)
  • No duplicate base tags found on any single page (Meta Pixel Helper would flag this)

Tag Firing

0 of 6 complete

  • PageView / base tag fires on page load (confirmed in Network tab or browser extension)
  • Conversion event fires ONLY on the confirmation / thank-you page (not on all pages)
  • Network requests return 200 or 204 status (not 4xx or blocked)
  • Events appear in platform real-time / test events tool within 60 seconds
  • Conversion value and currency are passed correctly (if applicable)
  • No JavaScript errors in the browser console (F12 → Console tab) on the conversion page

GTM Health Check

0 of 4 complete

  • GTM container has been published (not just saved / in preview)
  • Tags verified in GTM Preview mode before publishing
  • GTM container snippet is present in the website source (search for "googletagmanager.com/gtm.js" in page source)
  • No leftover test / dev tags in GTM that are still active on production

Platform-Specific Checks

0 of 5 complete

  • Google: GA4 DebugView shows the conversion event with correct parameters (value, currency, transaction_id)
  • Meta: Events Manager Test Events confirms the Purchase (or custom event) with no warnings
  • TikTok: TikTok Pixel Helper shows green status with correct event name on confirmation page
  • LINE: Network tab confirms insight.line.me request fires on conversion page with correct Tag ID
  • Conversion window (lookback period) is set correctly in the platform (e.g., 30-day click, 1-day view)
✓

Screenshot your validation evidence (Tag Assistant results, Network tab requests, Events Manager test events) and attach them to the campaign launch documentation. This protects you if a client later disputes conversion data.
